CALMAP - VERSION FINALE ET LANCEMENT DU SITE
============================================

Version finale a evaluer :

  branch_v4

Depot GitHub :

  https://github.com/calmap-team/calmap.git

La version finale du projet est disponible sur la branche branch_v4.
Il faut donc recuperer cette branche, et non une autre branche du depot.


1. RECUPERER LA VERSION FINALE
------------------------------

Depuis un dossier de travail au choix, lancer :

  git clone --branch branch_v4 --single-branch https://github.com/calmap-team/calmap.git calmap-branch-v4

Puis entrer dans le dossier du projet :

  cd calmap-branch-v4

Pour verifier que la bonne branche est utilisee :

  git branch

La branche active doit etre :

  branch_v4


2. PREREQUIS
------------

Avant de lancer le site, verifier que la machine dispose de :

- Python 3.11 ou plus ;
- Node.js 18 ou plus, avec npm ;
- Git ;
- PowerShell ou Invite de commandes Windows ;
- Terminal sur macOS.


3. LANCEMENT RECOMMANDE
-----------------------

Depuis la racine du projet, lancer :

  .\start.bat

Ce script :
- verifie Python ;
- installe les dependances Python si besoin ;
- verifie que le graphe de donnees existe ;
- installe les dependances frontend si besoin ;
- construit le frontend ;
- lance le serveur web.

Une fois le serveur lance, ouvrir dans un navigateur :

  http://127.0.0.1:8000/

Il faut garder le terminal ouvert pendant la demonstration.
Pour arreter le site, faire Ctrl+C dans le terminal.


4. LANCEMENT RAPIDE SI LES DEPENDANCES SONT DEJA INSTALLEES
-----------------------------------------------------------

Depuis la racine du projet :

  powershell -ExecutionPolicy Bypass -File .\start.ps1 -SkipInstall

Cette commande evite de reinstaller les dependances et lance directement
la construction du frontend puis le serveur.


5. LANCEMENT SUR MACOS
----------------------

Sur macOS, ouvrir l'application Terminal, se placer dans la racine du projet,
puis lancer les commandes suivantes :

  python3 -m pip install -r requirements.txt
  cd frontend
  npm install
  npm run build
  cd ..
  python3 -m uvicorn backend.main:app --host 127.0.0.1 --port 8000

Quand le serveur affiche qu'il est lance, ouvrir le site avec :

  open http://127.0.0.1:8000/

Il faut garder le terminal ouvert pendant la demonstration.
Pour arreter le site, faire Ctrl+C dans le terminal.

Si la commande python3 ne fonctionne pas, verifier l'installation de Python
ou essayer :

  python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000


6. LANCEMENT EN MODE DEVELOPPEMENT
----------------------------------

Terminal 1 - Backend FastAPI :

  python -m pip install -r requirements.txt
  python -m uvicorn backend.main:app --reload --port 8000

Terminal 2 - Frontend Vite :

  cd frontend
  npm install
  npm run dev

Dans ce mode, le frontend est accessible a l'adresse affichee par Vite,
souvent :

  http://localhost:5173/

Le backend reste disponible sur :

  http://localhost:8000/


7. NOTES IMPORTANTES
--------------------

- Le fichier data\graph.pkl est fourni avec la version finale.
- Si ce fichier manque, lancer depuis la racine :

  python pipeline\build_graph.py

- La documentation technique de l'API est disponible quand le backend tourne :

  http://localhost:8000/docs

- Si la commande uvicorn n'est pas reconnue sous Windows, utiliser :

  python -m uvicorn backend.main:app --reload --port 8000
